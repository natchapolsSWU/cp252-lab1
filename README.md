## TOR: ระบบบริหารจัดการที่จอดรถ (Car Park Management System) <br>
1. หลักการและเหตุผล
ปัจจุบันการบริหารจัดการที่จอดรถจำเป็นต้องมีระบบที่ช่วยเพิ่มความสะดวกให้ผู้ใช้งาน ลดปัญหาที่จอดไม่เพียงพอ และเพิ่มประสิทธิภาพในการจัดการพื้นที่จอดรถ ระบบบริหารจัดการที่จอดรถ (Car Park Management System) จึงถูกพัฒนาขึ้นเพื่อช่วยให้ผู้ใช้งานสามารถตรวจสอบ จอง และจัดการที่จอดรถได้อย่างเป็นระบบ รวมถึงช่วยให้ผู้ดูแลสามารถคำนวณค่าธรรมเนียมและบริหารสถานะที่จอดรถได้อย่างถูกต้องและมีประสิทธิภาพ
2. วัตถุประสงค์ของโครงการ
เพื่อพัฒนาระบบที่สามารถตรวจสอบสถานะที่จอดรถได้แบบเรียลไทม์
เพื่ออำนวยความสะดวกให้ผู้ใช้สามารถจองที่จอดรถล่วงหน้าได้
เพื่อให้ระบบสามารถคำนวณค่าธรรมเนียมการจอดรถได้อย่างถูกต้อง
เพื่อเพิ่มประสิทธิภาพในการบริหารจัดการพื้นที่จอดรถ และลดความซ้ำซ้อนในการใช้งาน
3. ขอบเขตของระบบ (Scope of Work)
3.1 Functional Requirements
ระบบต้องมีความสามารถดังต่อไปนี้
สามารถตรวจสอบสถานะที่จอดรถ (ว่าง/ไม่ว่าง) ได้
สามารถจองที่จอดรถล่วงหน้าได้
สามารถยกเลิกการจองที่จอดรถได้
สามารถคำนวณค่าธรรมเนียมการจอดรถตามระยะเวลาใช้งานได้
ระบบต้องสามารถคืนสถานะที่จอดรถเป็น “ว่าง” ได้โดยอัตโนมัติเมื่อสิ้นสุดการใช้งานหรือยกเลิกการจอง
3.2 Non-Functional Requirements
ระบบต้องมีคุณสมบัติดังต่อไปนี้
มีความถูกต้องของข้อมูล (Data Accuracy)
มีความพร้อมใช้งานของระบบ (Availability)
ใช้งานง่ายและเป็นมิตรกับผู้ใช้ (Usability)
รองรับผู้ใช้จำนวนมากพร้อมกันได้ (Scalability)
มีความปลอดภัยของข้อมูลผู้ใช้และข้อมูลการจอง (Data Security)
4. คุณสมบัติของระบบโดยรวม
ระบบต้องสามารถใช้งานผ่านอุปกรณ์ดิจิทัลได้สะดวก
มีการจัดการข้อมูลที่จอดรถ การจอง และการคิดค่าธรรมเนียมอย่างเป็นระบบ
มีความเสถียรและปลอดภัยต่อข้อมูลผู้ใช้งาน
5. ผลลัพธ์ที่คาดหวัง
ผู้ใช้สามารถค้นหาและจองที่จอดรถได้สะดวกมากขึ้น
ผู้ดูแลสามารถบริหารจัดการพื้นที่จอดรถได้อย่างมีประสิทธิภาพ
ลดปัญหาความซ้ำซ้อนและความผิดพลาดในการจัดการที่จอดรถ
